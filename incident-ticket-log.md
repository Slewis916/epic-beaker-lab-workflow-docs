## Ticket #001 — LIS Interface Downtime — Results Not Transmitting to Epic Beaker

| Field | Detail |
|---|---|
| Ticket ID | LAB0001-INC-001 |
| Priority | P2 |
| Reported By | Medical Laboratory Scientist |
| Date | June 2026 |

**Description:**
Laboratory results failed to transmit from LIS to Epic Beaker. Orders resulted on 
instruments but results not appearing in Beaker for provider review, causing potential 
delays in patient care.

**Root Cause:**
LIS system downtime interrupting HL7 interface transmission between LIS and Epic Beaker. 
Results queued on LIS side pending interface restoration.

**Immediate Action:**
Identified transmission failure. Escalated to IT application support team for interface 
investigation and restoration.

**Resolution:**
IT team restored LIS interface connection. Queued results transmitted to Epic Beaker upon 
restoration. Results reviewed and released to provider charts.

**Prevention Recommendation:**
Configure interface monitoring alerts to notify application support team automatically 
when LIS to Beaker transmission fails, reducing detection and escalation time.

## Ticket #002 — Result Transmission Failure — Specimen Not Received or Incorrect Container ID

| Field | Detail |
|---|---|
| Ticket ID | LAB0001-INC-002 |
| Priority | P2 |
| Reported By | Medical Laboratory Scientist |
| Date | June 2026 |

**Description:**
Laboratory result failed to transmit to Epic Beaker. Investigation revealed either the 
specimen was not received into Epic Beaker or the test was attached to an unreceived 
container ID. External facility specimens are particularly susceptible to this issue as 
packing list processing generates multiple container IDs per specimen.

Example: Specimen IDs LAB0001-26Y-000VI001.1, LAB0001-26Y-000VI001.2, and 
LAB0001-26Y-000VI001.3 were generated for a single specimen. Container IDs .1 and .2 
were received but .3 was not. The test was attached to container ID .3 preventing 
transmission.

**Root Cause:**
Specimen not received in Epic Beaker OR test attached to unreceived container ID. 
External facility specimens arriving via packing list generate multiple container IDs — 
not all containers consistently received on arrival.

**Immediate Action:**
Reviewed all container IDs associated with the order. Identified unreceived container ID. 
Received all outstanding container IDs in Epic Beaker. Confirmed all instances received 
before retransmitting.

**Resolution:**
All container IDs received in Epic Beaker. Result transmitted successfully to provider 
chart via platform transmit to LIS function.

**Prevention Recommendation:**
Implement verification step during external specimen accessioning to confirm all container 
IDs generated by packing list processing are received in Epic Beaker before specimens 
proceed to testing.

## Ticket #003 — Auto-Verification Failure — Manual Verification Required

| Field | Detail |
|---|---|
| Ticket ID | LAB0001-INC-003 |
| Priority | P3 |
| Reported By | Medical Laboratory Scientist |
| Date | June 2026 |

**Description:**
Results meeting expected auto-verification criteria failed to auto-verify and release to 
provider chart. Results held in queue requiring manual review and verification by 
laboratory staff despite appearing within normal parameters.

**Root Cause:**
Root cause not identified at end-user level. Possible causes include CER rule 
configuration error, QC failure flag, delta check trigger, or interface timing issue 
preventing auto-verification criteria from being evaluated correctly.

**Immediate Action:**
Manually reviewed and verified held results to prevent delays in patient care and 
provider notification.

**Resolution:**
Results manually verified and released to provider charts. Incident documented and 
escalated to application support team for CER rule review and investigation.

**Prevention Recommendation:**
Application support team to audit CER rule configuration for affected test to identify 
intermittent auto-verification failure trigger. Consider adding monitoring alert for 
unusual volume of manually verified results as indicator of CER rule malfunction.

## Ticket #004 — Incorrect Laboratory Test Ordered — Cancellation and Reorder Required

| Field | Detail |
|---|---|
| Ticket ID | LAB0001-INC-004 |
| Priority | P3 |
| Reported By | Medical Laboratory Scientist |
| Date | June 2026 |

**Description:**
Provider placed incorrect laboratory test order using wrong test code. Existing order 
canceled and correct test ordered to ensure accurate routing, processing, and billing.

**Root Cause:**
Order entry confusion between similarly named tests or incorrect test selected from 
order search results in Epic. Provider unfamiliar with correct order code for requested 
test.

**Immediate Action:**
Canceled incorrect order. Placed correct order with verified test code. Used instrument 
trigger to send corrected order to instrument where applicable. If instrument trigger 
was missed — manually added correct test through platform interface.

**Resolution:**
Correct test ordered and added to specimen on platform. Result transmitted to LIS and 
released to provider chart.

**Prevention Recommendation:**
Review order search result prioritization in Epic to surface most commonly ordered tests 
first. Consider adding clinical decision support alerts for commonly confused test orders 
to reduce incorrect order entry.

## Ticket #005 — Specimen Misrouted to Incorrect Laboratory

| Field | Detail |
|---|---|
| Ticket ID | LAB0001-INC-005 |
| Priority | P2 |
| Reported By | Medical Laboratory Scientist |
| Date | June 2026 |

**Description:**
Specimen physically routed to incorrect laboratory department following collection. 
Receiving laboratory either tracked specimen and returned it or specimen location unknown 
requiring active search across multiple laboratory departments. Resulted in delayed test 
processing and potential patient care impact.

**Root Cause:**
Two identified causes — incorrect routing label generated at collection or accessioning, 
or specimen manually delivered to wrong department without system verification. Receiving 
lab handling inconsistent — some departments track and return misrouted specimens, others 
do not resulting in specimen hunt.

**Immediate Action:**
Contacted receiving laboratories to locate specimen. Searched Epic Beaker for last known 
specimen status and tracking information. Coordinated physical search across laboratory 
departments when specimen not tracked by receiving lab.

**Resolution:**
Specimen located through interdepartmental communication or physical search. Delivered 
to correct laboratory and processed. Order updated in Epic Beaker to reflect correct 
processing laboratory and status.

**Prevention Recommendation:**
Review specimen routing label configuration in Epic to ensure correct destination 
laboratory prints at collection. Standardize specimen tracking and return procedure 
across all laboratory departments to reduce search time when misrouting occurs.
